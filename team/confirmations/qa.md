## (a) 테스트 대상 핵심 로직 선정

아래 5개 영역을 최우선 테스트 대상으로 선정한다. 오류 발생 시 사용자 신뢰·아동 안전·법적 리스크에 직결되는 순서로 정렬했다.

### 1. 임신 주수 계산 (WeekCalculator)
- **근거**: LMP 또는 EDD를 기준으로 임신 주차(D+주차), D-day, 삼분기 구분을 자동 산출한다. 오산 시 잘못된 검진 알림·태아 가이드가 노출되어 의료적 위해 가능성이 있다.
- **주요 입력**: `lmpDate`, `eddDate`, 기준일(오늘)
- **주요 출력**: 주차(Int), 일차(Int), 삼분기(1/2/3), D-day(Int)
- **엣지**: EDD만 입력한 경우, 다태아(쌍둥이) 케이스, 예정일 당일/초과 케이스, 유효하지 않은 날짜(미래 LMP 등)

### 2. 아이 월령 계산 (AgeCalculator)
- **근거**: 예방접종 스케줄, 정부지원금 자격, 성장 백분위, 성장 카드 D+일 표시 등 앱 전반에서 사용된다. 오산은 접종 누락 알림 미발송 등 아동 안전 이슈로 직결된다.
- **주요 입력**: `birthDate`, 기준일(오늘)
- **주요 출력**: D+일(Int), 개월수(Int), 나이(Int)
- **엣지**: 윤년 출생, 월말(1월 31일 출생 → 2월 계산), 기준일이 birthDate 이전인 경우(소급 기록)

### 3. Pregnancy → Child 데이터 승계 (PregnancyToChildMigration)
- **근거**: CLAUDE.md 및 SPEC.md 1.3절이 명시하는 "끊김 없는 여정"의 핵심 로직. 태명→이름, EDD→birthDate, 배 사진 타임라인→성장 사진 연속이 정확하게 이어져야 한다. 실패 시 데이터 손실 또는 중복 프로필 생성 위험.
- **주요 입력**: `Pregnancy` 엔티티(lmpDate, eddDate, nickname, fetusCount, status)
- **주요 출력**: `Child` 엔티티(birthDate 확정, 임신 기록 아카이브 보존)
- **엣지**: 쌍둥이 전환(2개 Child 생성), 상실 상태(`status = .paused`) 중 전환 시도, 예정일과 실제 출산일 다를 때, 태명 없는 경우

### 4. 예방접종 스케줄 자동 생성 (VaccineScheduleGenerator)
- **근거**: 질병관리청 API 기반으로 월령별 접종 일정을 자동 생성한다. D-7·D-1·당일 알림과 연동되며, 기한 초과 경고도 여기서 파생된다. 오산은 아동 보건 위해.
- **주요 입력**: `birthDate`, 질병관리청 백신 데이터셋(로컬 번들)
- **주요 출력**: `[VaccineRecord]`(scheduledDate, vaccineId, status)
- **엣지**: 조기 출생아 교정 월령 미지원 시 처리, 이미 과거에 맞은 접종 표시, 다자녀 간 일정 독립성, 번들 데이터 버전 갱신 시 기존 레코드 처리

### 5. WHO 성장 백분위 매핑 (GrowthPercentileMapper)
- **근거**: 키·몸무게·머리둘레를 WHO 공개 데이터셋에 대응해 백분위를 산출하고 성장 카드·차트에 표시한다. 성별·월령 테이블을 정확히 참조해야 하며, 잘못된 수치가 의료적 판단에 영향을 줄 수 있다.
- **주요 입력**: `height(cm)`, `weight(kg)`, `headCircumference(cm)`, `ageInMonths(Int)`, `gender`
- **주요 출력**: `heightPercentile(Double)`, `weightPercentile(Double)`, 정성적 안심 메시지(String)
- **엣지**: 극단값(측정 오류 수준), 성별 미선택(non-binary 처리), WHO 테이블 경계 월령(0개월, 60개월), cm↔inch 단위 전환 후 역산 정확도

## (b) 테스트 분류 제안

| 테스트 유형 | 적용 대상 | 이유 |
|---|---|---|
| **단위(Unit) — XCTest** | 주차/월령 계산, Pregnancy→Child 승계, 예방접종 스케줄 생성, 백분위 매핑, 뱃지 조건 판정, 지원금 자격 판정 | 입력→출력이 명확한 순수 로직. 빠르고 결정적이어야 함. CoreData 의존성은 in-memory store로 격리 |
| **단위(Unit) — in-memory CoreData** | Pregnancy·Child·GrowthRecord·VaccineRecord CRUD, 전환 로직 | CoreData 스택을 실제 디스크 없이 격리 테스트 |
| **위젯(Widget) — WidgetKit Preview** | 오늘의 할 일 위젯, 아이 요약 위젯, 주변 응급 위젯 | 데이터가 없거나 희소할 때의 fallback 렌더링, Dynamic Type 확대 시 레이아웃 깨짐 여부 |
| **통합(Integration) — XCTestCase + URLProtocol Stub** | 질병관리청 API 파싱, 카카오맵 API 응답 처리, 심평원 API 처리 | 실제 API를 stub으로 교체해 네트워크 독립성 확보 |
| **통합(Integration) — CloudKit mock** | 가족 공유 동기화, 아이 프로필 공유, 성장 기록 동기화 | CloudKit은 시뮬레이터에서 제약이 있으므로 CloudKitSyncEngine을 Protocol 추상화 후 mock 치환 |
| **접근성 감사 — AXAudit** | 홈 화면, 예방접종 화면, 응급 모드 화면 | Dynamic Type·VoiceOver 레이블·색 대비 자동 감사 (XCUITest + `XCTAssertElement`) |
| **UI(Snapshot) — Point-Free SnapshotTesting 또는 XCTest** | 성장 카드 공유 이미지, 뱃지 프로필 카드, 주간 리포트 | ImageRenderer 출력 픽셀 회귀 방지 |

> **우선순위**: Unit(계산 로직) > Unit(CoreData 전환) > 통합(API stub) > 접근성 감사 > 위젯·스냅샷
>
> v1 MVP 8주 타임박스 기준, 5주차(예방접종 API 연동) 이전에 단위 테스트 뼈대를 완성해야 회귀 방지 효과가 있다.
