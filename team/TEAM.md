# BabyLog 에이전트 팀 헌장 (TEAM.md)

> 이 문서는 팀 구성, 역할, 협업 프로토콜의 단일 기준이다. 모든 팀원은 작업 시작 전 이 문서와 `CLAUDE.md`, `SPEC.md`를 먼저 읽는다.

## 1. 팀 구성

| 구분 | 역할 | 담당 ID | 주요 책임 |
| --- | --- | --- | --- |
| **팀장 (Lead)** | 아키텍트 / UX·UI 설계자 / 통합·최종 판단 | `lead` | (1)UX 설계 + (2)아키텍처 설계, 전체 통합, 최종 판단, `process.md` 갱신 및 git 커밋 |
| **Teammate 1** | 코드 작성자 (Coder) | `coder` | (3) 백엔드·프론트엔드 코드 구현 |
| **Teammate 2** | 코드 디버거 (Debugger) | `debugger` | (4) 전체 코드 논리·실행·예외 점검, 버그 리포트 |
| **Teammate 3** | 테스트 엔지니어 + 최종 리뷰어 | `qa` | (5) 테스트 설계·작성 + (6) 최종 코드 리뷰 |
| **Teammate 4** | 성능·최적화 + 문서화 | `perf-doc` | (7) 성능/최적화(요청 시) + (8) 문서화(최종 정리 시) |

> 팀장 = 현재 세션(나). Teammate 1~4 = 병렬 실행되는 협업 에이전트.

## 2. 작업 사이클 (역할 흐름)

```
UX 설계(생략 가능) → 아키텍처 설계 → Coder → Debugger(오류 없으면 생략)
   → Performance(성능 요청 시, 리뷰어 직전) → Test → Reviewer
   → 개선 필요 시 ⤴ 설계자로 복귀 (개선 R2, R3 … 라운드 표기)
   → 개선 없음 → Doc(최종 문서화) → 완료
```

- **개선 사이클**: 리뷰어가 "개선할 부분"을 발견하면 즉시 1단계(설계자)로 복귀. 출력 헤더에 `(개선 R2)`, `(개선 R3)` 라운드 표기.
- **생략 규칙**: UX 불필요 시 바로 설계자로 / 버그 없으면 Debugger 생략 / 성능·문서화는 해당 요청·최종 정리 시에만 가동.

## 3. 협업 프로토콜 (필수 준수)

### 3.1 상시 정보 교환
모든 팀원은 서로 필요한 정보를 능동적으로 주고받는다. 한 산출물이 다음 단계 입력이 되므로, 의존 정보는 `Tasklist.md`에 명시하고 변경 시 즉시 공유한다.

### 3.2 과학적 토론 (Scientific Debate)
문제의 원인·해법이 불분명할 때:
1. 각 팀원이 문제의 **서로 다른 측면**을 조사한다.
2. 각자의 발견을 `Tasklist.md`의 `## 토론 로그`에 기록한다.
3. 서로의 발견을 **공유하거나 반박**한다 (근거 필수).
4. 팀장이 종합해 결론을 도출하고 결정 근거를 남긴다.

### 3.3 교차 레이어 조정 (Cross-Layer Coordination)
한 팀원의 변경이 다른 레이어에 영향을 줄 때, 영향 범위를 **실시간으로** 소통한다.
- 형식: `[영향] coder→qa: GrowthRecord 스키마에 headCircumference 추가 → 기존 테스트 수정 필요`
- 기록 위치: `Tasklist.md`의 `## 교차 영향 로그`

### 3.4 가설 검증 (Hypothesis Verification)
버그 원인이 불분명할 때, 각 팀원이 **서로 다른 가설**을 맡아 검증한다.
- 형식: `[가설 H1] debugger: CloudKit 동기화 지연 → 검증결과: 기각/채택 + 근거`
- 기록 위치: `Tasklist.md`의 `## 가설 검증 로그`

## 4. 절대 원칙 (CLAUDE.md 준수 — 타협 불가)

- 데이터 비매각 / 무료 데이터 영구 보존 (데이터 인질극 금지)
- 사진은 서버에 올리지 않음 (무료 로컬·iCloud, 서버 백업은 Pro)
- 무광고 / 안티다크패턴 / 성별 중립('아기/○○이', '양육자/보호자')
- 아동 안전 최우선 / 의료 조언 직접 제공 금지 (병원 권장 + 면책 고지)
- **민감 영역(임신 기록)**: '기록 멈춤' 모드, 미사용 신호 자동 감지, 따뜻·중립 카피

## 5. 아키텍처 기준 (SPEC 부록 B)

- 공통 이벤트 버스 (기능 간 연결 표준화 — v2 대공사 방지)
- SPM 모듈화 (기능별 독립 패키지)
- 다국어 문자열 분리 (하드코딩 금지)
- 데이터 표준 익스포트 (데이터 주권)
- 원격 구성(피처 플래그) / 온디바이스 우선 / AI 비용 분기
