## 공통 원칙

### 성별 중립 & 민감 영역 카피

- **호칭**: 기본값은 '○○님' / '양육자'. '맘' / '파파'는 사용자가 선택한 경우에만 표시.
- **가족 범주**: 아빠·조부모·입양·위탁·조손가정을 포함하는 언어. "엄마/아빠" 하드코딩 금지.
- **상실 배려**: 임신 관련 카피는 항상 상실 가능성을 전제하고 작성. 닦달·재촉 어조 절대 금지.
- **또래 비교**: 기본은 정성적 안심 메시지("또래와 비슷하게 잘 크고 있어요"). 정밀 백분위는 옵트인.
- **의료 면책**: 모든 태아·건강 정보에 "일반 정보이며 의료 상담을 대체하지 않음" 명시.

### 강제 입력 0 원칙

어떤 화면에서도 사용자를 강제로 막는 필수 입력 단계는 존재하지 않는다. 모든 입력은 선택이며, 나중에 추가할 수 있어야 한다.

## 1. 온보딩 (`Onboarding`)

**목적**: 앱의 가치를 강요 없이 보여주고 개인화 기반을 수집. 게스트 둘러보기 우선.

**파일 참조**: `design/handoff/design_files/app/onboarding.jsx`  
**SPEC 참조**: 기능 9 온보딩 & 첫 사용자 경험

### 플로우 (5단계, 단계별 스킵 가능)

| 단계 | 화면명 | 핵심 내용 |
|---|---|---|
| 0 | 스플래시 | BabyLog 로고 + "우리 동네 육아의 모든 것" 한 줄 메시지. 시작하기 / 이미 계정이 있어요 2개 버튼. |
| 1 | 게스트 가치 미리보기 | 가입 없이 현재 위치 기준 영업 중 소아과 즉시 표시. "가입하지 않아도 둘러볼 수 있어요" 안내. |
| 2 | 기록 밀도 선택 | '가볍게 / 꼼꼼히' 1문항. 이후 UI 프리셋 결정. 나중에 변경 가능. |
| 3 | 아이/임신 등록 | 임신·출산 분기 선택 → 생년월일(출산) 또는 출산 예정일(임신) 입력 → D+일·예방접종 타임라인 또는 임신 주차·산전검진 자동 생성. 이름은 선택. 나중에 할게요 버튼 항상 노출. |
| 4 | 권한 요청 (프리퍼미션) | 위치(근처 소아과 맥락)·알림(접종일 맥락) 순서로 이유와 함께 요청. "나중에 설정할게요" 항상 제공. |

### 임신 / 출산 분기

- 단계 3에서 '출산했어요' / '임신 중이에요' 토글로 분기.
- **출산** 선택: 아이 생년월일 입력 → D+일·예방접종 타임라인 자동 생성.
- **임신** 선택: 출산 예정일 입력 → 임신 주차·산전검진 일정 자동 생성. 임신 모드로 앱 진입.

### 핵심 액션

- 모든 단계에서 스킵 가능 (강제 입력 0).
- 게스트 데이터는 로컬 저장 → 가입 시 계정으로 마이그레이션.
- 위치 권한 거부 시 수동 지역 선택으로 핵심 가치 유지.

### 빈 상태

- 아이 등록 없이 완료 시 홈은 "아이를 등록하면 맞춤 서비스를 시작해요" 안내 카드 표시.

### 예외

- 위치 권한 거부: 수동 지역 선택 fallback 제공.
- 알림 권한 거부: 설정에서 나중에 요청 가능 안내.
- 네트워크 없음: 게스트 소아과 목록은 캐시 또는 "네트워크가 필요해요" 안내.

## 2. 홈 (`HomeScreen`)

**목적**: 매 세션 첫 화면. "오늘의 한 장면"으로 리텐션 유지. 진입점 역할, 데이터 본체는 각 탭에 있음.

**파일 참조**: `design/handoff/design_files/app/home.jsx`  
**SPEC 참조**: 기능 10 홈 화면 & 리텐션 루프

### 육아 모드 (AppMode.baby)

| 영역 | 내용 |
|---|---|
| 상단 헤더 | 인사말 + 응급 버튼(우상단 빨간 pill). 레이아웃 A: 최근 사진 히어로(188pt 높이, 하단 이름·D+일 오버레이). 레이아웃 B/C: 아바타+이름+D+일 컴팩트 헤더. |
| 다자녀 전환 칩 | 상단 아이 아바타 칩 탭으로 즉시 전환. 아이 추가(+) 버튼 포함. |
| 우선순위 단일 카드 | 우선순위 엔진(룰 베이스)이 선정한 '지금 가장 중요한 것' 1개만 크게. 골드 그라데이션 배경(#FBF1DC→#F7E7C4). D-day 숫자 + 주요 액션 버튼 포함. |
| 하위 모듈 | 기록 권유 / 또래 이야기 / 1년 전 오늘 추억 / 동네 소식 / 이번 달 육아비 등 3~4개. 카드당 단일 액션. |

**우선순위 엔진 순위**:
1. 예방접종 임박 (D-1, 당일)
2. 정부지원금 신청 마감 임박
3. 응급 상황 감지 (야간 + 최근 검색)
4. 1년 전 오늘 추억 카드
5. 동네 새 매물 / 크루 활동
6. 기록 권유 (부드럽게 — "○○의 오늘이 궁금해요")

**레이아웃 3안** (원격 구성으로 A/B/C 전환 가능):
- A: 히어로 사진 상단 + 스크롤 모듈
- B: 컴팩트 헤더 + 2×2 타일 그리드 + 모듈
- C: 컴팩트 헤더 + 최근 기록 인라인 타임라인

### 임신 모드 (AppMode.pregnancy)

| 영역 | 내용 |
|---|---|
| 히어로 카드 | 핑크 그라데이션(#FBE6EE→#F6D6E4). 과일 이모지로 태아 크기 시각화. 임신 삼분기·주차 뱃지. D-day(출산까지) 숫자. |
| 우선순위 카드 | 산전검진·검사 임박 알림. 핑크 톤 골드카드. |
| 주차 가이드 | "N주차 태아 이야기" 카드. 발달 상황 안내. 의료 면책 포함. |
| 하위 모듈 | 동네 소식·육아비 모듈. |

### 핵심 액션

- 응급 버튼 탭 → 응급 모드 전환 (풀스크린 다크).
- 우선순위 카드 탭 → 해당 전용 탭·상세 딥링크.
- 기록 권유 카드 → 빠른 기록 시트 오픈.

### 빈 상태

- 기록 0개: "사진 한 장이면 시작할 수 있어요" + FAB 강조 애니메이션.
- 예방접종 데이터 없음: 접종 등록 유도 카드.

### 예외

- 로딩: 카드 영역 skeleton 플레이스홀더.
- 네트워크 없음: 로컬 데이터 우선 표시, 동네 소식 카드는 "연결이 필요해요" 상태.
- 상실 상태 (`Pregnancy.status == .loss`): 임신 우선순위 카드·주차 가이드 카드 완전히 숨김. 기록 멈춤 모드 안내 카드 노출.

## 3. 기록 탭 — 육아 모드 (`RecordScreen`, AppMode.baby)

**목적**: 아이의 전체 타임라인. 임신부터 성장까지 끊김 없는 사적 서사.

**파일 참조**: `design/handoff/design_files/app/record.jsx`  
**SPEC 참조**: 기능 2 성장 기록 & 육아 일지

### 세그먼트 구조

상단 3-세그먼트 버튼: **타임라인 | 성장차트 | 예방접종**

#### 타임라인 세그먼트

- 날짜별 섹션 헤더 + 기록 카드 목록.
- 카드 유형: 사진(전체 너비 이미지 + 캡션), 성장 측정(키·몸무게 수치), 예방접종(접종명·병원).
- 이정표 뱃지: 골드 amber 배경으로 카드 좌상단 표시 ("첫 걸음마" 등).
- 최하단: "○○의 N번째 순간이 기록되었어요" 누적 카운트.
- 태아 시절 배 사진은 "태아 시절" 섹션으로 타임라인에 연속 노출 (Phase 3 구현).

#### 성장차트 세그먼트

- 안심 메시지 카드 최상단 (primary-tint 배경): "또래와 비슷하게 잘 크고 있어요". 백분위 숫자는 옵트인 확장.
- Swift Charts로 WHO 성장 곡선 오버레이 + 아이 실측치 라인.
- 측정 항목 토글 칩: 몸무게 / 키 / 머리둘레.
- 하단 통계 3칸: 현재 수치 / WHO 백분위 / 최근 변화량.

#### 예방접종 세그먼트

- 임박 접종 상단 강조 카드 (골드 그라데이션, D-day 숫자 크게).
- 데이터 소스: 질병관리청 예방접종도우미 API.
- 접종별 상태: 완료(mint 뱃지) / 임박(amber 뱃지, D-N) / 예정(날짜).
- 미접종 기한 초과 시 danger 색상 경고 표시.

### 공유 버튼

헤더 우측 공유 아이콘 → 성장 카드 공유 화면으로 이동.

### 핵심 액션

- FAB(빠른 기록) → 빠른 기록 시트.
- 성장차트 데이터 점 탭 → 측정일·수치 상세.
- 예방접종 카드 탭 → 예약 안내 / 완료 체크.

### 빈 상태

- 타임라인 0개: "첫 기록을 남겨볼까요?" + FAB 강조.
- 성장 측정 없음: 측정 입력 유도 안내.
- 예방접종 등록 전: 아이 생년월일 기반 스케줄 자동 생성 안내.

### 예외

- 사진 EXIF 없음 (카톡 수신 등): "촬영일 직접 선택" fallback.
- 대량 사진 스크롤: 썸네일 사전생성·캐시로 성능 대응 (Phase 3 구현).
- API 응답 없음: 로컬 번들 기본 스케줄 표시 + "최신 정보는 연결 후 확인" 안내.

## 4. 기록 탭 — 임신 모드 (`RecordScreen`, AppMode.pregnancy)

**목적**: 임신 기록 전용. 태아 성장·산모 기록·산전검진 통합.

**SPEC 참조**: 기능 1 임신 기록 & 태아 일지

### 세그먼트 구조

**주차 가이드 | 산모 기록 | 산전검진**

#### 주차 가이드 세그먼트

- 현재 주차 태아 크기 (과일 비유, 예: "레몬만 해요").
- 발달 상황 안내 텍스트. 의료 면책 항상 표시.
- 삼분기별 주의사항 안내.

#### 산모 기록 세그먼트

- 배 사진 D라인 타임라인 (날짜순). 출산 후 성장 사진과 연속.
- 체중 추이 차트 (Swift Charts, 권장 증가 범위 오버레이).
- 태동 카운터: 탭으로 횟수 기록.
- 증상 기록: 입덧·컨디션 등 선택 입력.

#### 산전검진 세그먼트

- 주수별 표준 산전검진 일정 자동 생성.
- 완료 체크, 검진일·병원·초음파 사진·메모 저장.
- 임박 검진 D-day 강조 카드.

### "출산했어요" 전환 버튼

- 상단 배너 또는 헤더에 항상 노출.
- 탭 → 출산 전환 플로우 (아이 이름 입력·생년월일 확정 → Pregnancy→Child 승계).
- 임신 기록은 "태아 시절" 아카이브로 영구 보존.

### 핵심 액션

- FAB → 임신 모드 액션: 태동 / 배 사진 / 메모.
- 출산 전환 버튼.

### 빈 상태

- 배 사진 없음: "첫 배 사진을 남겨볼까요?" 안내.

### 예외 (민감 영역)

- '기록 멈춤' 모드 진입 가능 (버튼 또는 상실 기록 시 자동). 진입 시 모든 주차 알림·태아 가이드 즉시 중단.
- `status == .loss`: 화면 전체를 따뜻하고 중립적인 아카이브 모드로 전환. 알림·가이드·권유 카드 완전 숨김. 기록 보존·삭제 선택권 제공.

## 5. 빠른 기록 시트 (`QuickRecordSheet`)

**목적**: 2탭 완료. FAB 탭 → 사진 선택 → 저장. 핵심 UX 지표(1mm 지표).

**파일 참조**: `design/handoff/design_files/app/record.jsx` (QuickRecordSheet 섹션)  
**SPEC 참조**: 기능 8.4 빠른 기록 프로필 버튼

### 구조

바텀시트 (slideUp 0.3s, 최대 높이 92%).

| 영역 | 내용 |
|---|---|
| 사진 드롭존 | 전체 너비 사진 플레이스홀더. 탭 → PhotosUI 선택. 사진 없이도 저장 가능. |
| 이정표 칩 행 | 가로 스크롤. '첫 미소', '첫 뒤집기' 등 프리셋 6개 노출. 선택 시 골드 토글. |
| 저장 버튼 | "저장하기" LiquidButton. 탭하면 즉시 저장. |
| 자세히 펼치기 | "자세히 입력 ▾" 버튼으로 상세 모드 확장. |

### 자세히 모드 (확장)

- 한 줄 메모 텍스트 입력.
- 키(cm) / 몸무게(kg) 숫자 필드.
- "AI로 캡션 초안 만들기 · Pro" 버튼.

### 저장 후 보상

- 팝 애니메이션 (0.5s). "○○의 N번째 순간" 타이틀.
- "타임라인에서 보기" 버튼 → 기록 탭 딥링크.

### 모드별 액션 (QuickRecordFAB 스피드다이얼)

| AppMode | 액션 |
|---|---|
| 육아 모드 | 성장 측정 / 사진 / 메모 |
| 임신 모드 | 태동 기록 / 배 사진 / 메모 |

### 핵심 UX 지표

FAB 탭 → 사진 선택 → 저장 완료: **2탭**. 이 경로에 강제 입력 단계 0.

### 빈 상태

- 사진 없이 저장 버튼 탭 가능 (메모만도 기록).

### 예외

- 사진 권한 없음: PHPicker로 최소 권한 요청.
- 저장 실패: 인라인 에러 메시지 + 재시도.

## 6. 성장 카드 공유 (`ShareCardScreen`)

**목적**: 아이 사진 위에 성장 데이터를 오버레이한 인스타용 공유 카드 생성. 바이럴 핵심 기능.

**파일 참조**: `design/handoff/design_files/app/sharecard.jsx`  
**SPEC 참조**: 기능 2.4 성장 카드 공유

### 화면 구조

다크 배경 (`#15110E`) 전체 화면.

| 영역 | 내용 |
|---|---|
| 카드 미리보기 | 실시간 렌더 프리뷰. 선택된 비율·위치·데이터·블러·워터마크 즉시 반영. |
| 비율 선택 | 4:5 (피드) / 1:1 (정사각) / 9:16 (스토리). 칩 선택. |
| 데이터 위치 | 좌하(기본) / 우하 / 좌상 / 중하 / 없음. 아이 얼굴 가림 방지 목적. |
| 표시 데이터 | 키 / 몸무게 / 월령·D+day / 또래 백분위 / 이정표 항목별 on/off 칩. |
| 프라이버시 컨트롤 | 얼굴 가리기 (블러 오버레이) 토글. 워터마크 on/off (무료: 항상 포함, Pro: 제거 가능). |
| 공유 버튼 | "공유하기" LiquidButton → `UIActivityViewController` (인스타·카톡 등). |

### 데이터 오버레이 구성

- 배경: 선택한 아이 사진 (전체 화면).
- 스크림: 하단(또는 상단) 그라데이션 — 어떤 사진에도 텍스트 가독성 확보.
- 데이터 위치: posStyle로 동적 배치.
- 브랜드 워터마크: 우상단 BabyLog 로고 (무료는 항상 표시, Pro 제거 가능).

### 얼굴 블러

- 블러 토글 ON: 얼굴 추정 영역에 반투명 원형 + backdropFilter 블러 오버레이.
- Phase 3에서 Core ML 온디바이스 얼굴 감지로 자동화 예정.

### 기술 구현

- `ImageRenderer` (SwiftUI 뷰 → UIImage 변환).
- 공유: `UIActivityViewController`.
- 카드 레이아웃 데이터 모델 분리 (`ShareCard` 엔티티) — 향후 템플릿 추가 용이.

### 수익 연계

- 무료: 워터마크 포함 → 자연 바이럴 (친구가 보고 앱 유입).
- Pro: 워터마크 제거 가능 → Pro 과금 동기.

### 핵심 액션

- 비율·위치·데이터·프라이버시 설정 조합 → 미리보기 실시간 반영.
- 공유 버튼 탭 → 외부 공유.

### 빈 상태

- 성장 데이터 없음: 키·몸무게 데이터 항목 비활성화 (회색 칩).

### 예외

- 사진 없음: 사진 선택 유도 안내.
- 렌더링 실패: "이미지 생성에 실패했어요. 다시 시도해주세요."

## 7. 동네 탭 (`DongneScreen`)

**목적**: 위치 기반 3기능 통합. 시간대에 따라 우선 세그먼트 자동 조정.

**SPEC 참조**: 기능 8.3 동네 탭 내부 구성, 기능 3 주변 인프라, 기능 4 마켓, 기능 6 크루

### 세그먼트 구조

**주변 | 마켓 | 크루**

맥락 적응: 야간(일과 이후) → 주변 세그먼트 우선. 주간 → 마켓·크루 우선.

### 7-1. 주변 세그먼트

**목적**: 야간 소아과 찾기가 첫 다운로드 핵심 동기.

| 카테고리 | 필터 | 내용 |
|---|---|---|
| 소아과 | 현재 영업중 / 야간진료 / 공휴일진료 | 건강보험심사평가원 API + 카카오맵 보완 |
| 약국 | 현재 영업중 / 24시간 / 야간약국 | 동일 |
| 키즈카페 | 연령 0-2세 / 3-5세 / 6세+ | 카카오맵 로컬 API |
| 놀이터 | — | 공공데이터 |

**리스트 퍼스트** (지도는 느림). 영업정보에 "○분 전 확인" 신뢰뱃지 표시.

#### 응급 모드 (킬러 기능)

- 홈 응급 버튼 또는 주변 탭 상단 버튼 탭으로 진입.
- **풀스크린 다크** 레이아웃 (`emergencyBg: #15110E`).
- 필터 없이 지금 영업중·거리순 소아과 즉시 표시.
- 전화 버튼 최대 크기.
- 초기: "전화로 확인" 전면 배치 (데이터 공백 우회).
- 신뢰 장치: "○분 전 확인" 표시 + "지금 문 닫혔어요" 신고 기능.
- 응급 종료 후: "오늘 진료 기록 남길까요?" → 기록 기능 연결.

#### 빈 상태 / 예외

- 위치 권한 없음: 수동 지역 선택 fallback.
- API 응답 없음: "현재 정보를 불러올 수 없어요. 전화로 확인해주세요." + 지역 대표 번호.

## 8. 가계부 탭 (`BudgetScreen`, v2~)

**목적**: 육아비 전용 카테고리 + 정부지원금 알림. 자동 수집 대시보드 중심.

**SPEC 참조**: 기능 5 육아 가계부 & 정부지원금

| 영역 | 내용 |
|---|---|
| 정부지원금 배너 | 전면 배치. 아동수당·부모급여·첫만남이용권 등 신청 D-day. 신청 방법 안내·딥링크. |
| 월 대시보드 | 총 육아비 + 전월 대비. 카테고리별 도넛 차트. 막대 비중 바. |
| 지출 목록 | 날짜·카테고리·금액. 앱 내 거래 자동 수집. |
| 지출 입력 | 금액·카테고리·날짜 필수, 메모·사진(영수증) 선택. 다자녀 아이 태그. |

**입력 최소화 원칙**: 마켓 거래·구독 결제는 자동 가계부 반영. 수동 입력은 큰 지출 위주.

### 빈 상태

- 첫 달: "육아비를 기록하면 월별 추이를 볼 수 있어요" + 첫 입력 유도.
- 정부지원금 없음 (해당 없는 나이): "지금은 해당 지원금이 없지만, 시기가 되면 알려드릴게요."
